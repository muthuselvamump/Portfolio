2.Configure Two-Tier Architecture on AWS

  • A public-facing Application Load Balancer in this architecture directs client traffic to our web tier EC2 instances.
  • The Apache web servers on the web layer are configured to serve webpages and reroute PHP-written apps.
  • The application modifies data before delivering it to our web tier using a MySQL multi-AZ database.
  • At each tier, auto scaling groups are set for CPU utilization 70%, route 53, health checks.
  • NAT gateways, and load balancing are built in order to preserve the availability of this architecture.
